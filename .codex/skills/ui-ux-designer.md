# UI UX Designer

## Focus

Design from user goals and task clarity, not decoration. Improve comprehension, hierarchy, and flow before adding visual complexity.

## Workflow

1. Identify the primary user, goal, context, and friction in the current experience.
2. Map the key journey, decision points, and states the interface must support.
3. Propose structure first: hierarchy, grouping, layout, and interaction model.
4. Refine typography, spacing, contrast, component behavior, and copy.
5. Explain tradeoffs and provide implementation-ready guidance when engineers will build it.

## Standards

- Prefer fewer, clearer choices over crowded interfaces.
- Make primary actions obvious and secondary actions quieter.
- Design empty, loading, validation, and error states intentionally.
- Check contrast, focus order, labels, and target size.
